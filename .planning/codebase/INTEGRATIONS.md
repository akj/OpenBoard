# External Integrations

**Analysis Date:** 2026-04-27

## Chess Engine Integration

**Stockfish (UCI engine):**
- Protocol: UCI (Universal Chess Interface) — handled by `chess.engine` from the `chess` library
- Communication: subprocess via `chess.engine.popen_uci()` (async, runs in a background thread with its own event loop)
- Adapter: `openboard/engine/engine_adapter.py` — `EngineAdapter` class wraps the async UCI protocol with a thread-safe synchronous interface for the wxPython GUI
- Detection: `openboard/engine/engine_detection.py` — `EngineDetector` searches PATH and platform-specific locations (e.g. `/usr/bin`, `/usr/games`, Homebrew paths on macOS, `C:\Program Files\` on Windows)
- Manager: `openboard/engine/stockfish_manager.py` — `StockfishManager` provides install/update/status lifecycle; emits blinker signals for UI progress
- Downloader: `openboard/engine/downloader.py` — `StockfishDownloader` fetches Windows binaries from the GitHub Releases API (see GitHub API below)
- Supported options: any valid UCI option passed as `dict[str, Any]` to `EngineAdapter`; defaults to auto-detected path
- Automatic install: Windows only; Linux/macOS require system package manager

**Engine is optional:** the application starts without a Stockfish binary; engine features are simply unavailable.

## External HTTP APIs

**GitHub Releases API (Stockfish downloads — Windows only):**
- Endpoint: `https://api.github.com/repos/official-stockfish/Stockfish/releases/latest`
- Purpose: determine the latest Stockfish version tag and find the Windows binary download URL
- Client: Python stdlib `urllib.request.urlopen` with a `User-Agent: OpenBoard Chess GUI` header
- Timeout: 10 seconds for version fetch; 30 seconds for binary download
- Used by: `openboard/engine/downloader.py` (`StockfishDownloader.get_latest_version()`, `StockfishDownloader.download_and_install_latest()`)
- No authentication required (public API, unauthenticated)
- No API key or credentials needed

**No other external HTTP APIs are used at runtime.**

## File Formats Handled

**PGN (Portable Game Notation):**
- Read/write via `chess.pgn` (`chess` library)
- Used in: `openboard/controllers/chess_controller.py` (PGN import/export)
- Standard chess game notation format; no third-party PGN service is called

**FEN (Forsyth–Edwards Notation):**
- Parsed and generated by `chess.Board` throughout the codebase
- Passed to `EngineAdapter` as position strings; validated in `openboard/engine/engine_adapter.py`

**Polyglot Opening Book (.bin):**
- Read via `chess.polyglot.MemoryMappedReader`
- Managed by `openboard/models/opening_book.py` — `OpeningBook` class
- User-supplied `.bin` files loaded at runtime; no bundled book file in the repository
- Controller entry points: `openboard/controllers/chess_controller.py` (`load_opening_book`, `request_book_hint`)

**JSON (Keyboard Configuration):**
- Read/written via Python stdlib `json` + pydantic `TypeAdapter`
- Default config: `config/keyboard_config.json`
- Loader: `openboard/config/keyboard_config.py` (`load_keyboard_config_from_json`, `save_keyboard_config_to_json`)

## Data Storage

**Databases:** None — no database of any kind is used.

**File Storage:**
- Local filesystem only
- Engine binaries installed to `engines/stockfish/bin/` relative to CWD (configurable via `EngineSettings.engines_dir`)
- Version metadata written to `engines/stockfish/version.txt` and `engines/stockfish/metadata.json`
- Log files written to `~/openboard.log` (rotating, max 10 MB × 5 backups); configured in `openboard/logging_config.py`

**Caching:** None — no in-process or distributed cache layer.

## Authentication & Identity

**Auth Provider:** None — OpenBoard is a local desktop application with no user accounts, login, or authentication of any kind.

## Accessibility Integration

**accessible-output3 (screen reader bridge):**
- Library: `accessible-output3` (git source: `https://github.com/SigmaNight/accessible_output3`)
- Auto-detects available screen readers on the current platform (NVDA, JAWS, speech-dispatcher, etc.)
- Used directly in `openboard/views/views.py` via `accessible_output3.outputs.auto`
- Not an HTTP API; communicates with the OS accessibility layer via platform-native IPC

**System TTS (Linux):**
- `speech-dispatcher` and `espeak-ng` are runtime dependencies on Linux (declared in `.build/installers/linux/rpm/openboard.spec` and installed by CI)
- Consumed indirectly through `accessible-output3`

## Monitoring & Observability

**Error Tracking:** None — no Sentry, Rollbar, or equivalent service is integrated.

**Logs:**
- Python `logging` module with `RotatingFileHandler`
- Destination: `~/openboard.log` (configurable)
- Format: `%(asctime)s - %(name)s - %(levelname)s - %(message)s`
- Configuration: `openboard/logging_config.py`

## CI/CD & Deployment

**Hosting:** GitHub — releases created on `v*.*.*` tags via `.github/workflows/release.yml` using `softprops/action-gh-release@v2`

**CI Pipeline:** GitHub Actions
- Main workflow: `.github/workflows/ci.yml` — triggers on push/PR to `main` or `build`
- Reusable workflows:
  - `.github/workflows/setup-lint.yml` — ruff check + format + ty type check (ubuntu-latest)
  - `.github/workflows/setup-test.yml` — pytest on ubuntu-latest, windows-latest, macos-latest
  - `.github/workflows/setup-validate.yml` — build validation
  - `.github/workflows/setup-build.yml` — PyInstaller build per platform
  - `.github/workflows/setup-build-installer.yml` — DEB, RPM, DMG, Inno Setup
  - `.github/workflows/setup-validate-executable.yml` — smoke tests on built executables
- Release workflow: `.github/workflows/release.yml` — full build → validate → installer → GitHub Release pipeline

**Artifact distribution:** GitHub Release assets (`.deb`, `.rpm`, `.dmg`, `.exe` installer, `.tar.gz`/`.zip` portable archives, `SHA256SUMS`)

## Webhooks & Callbacks

**Incoming:** None — no webhook endpoints exist; this is a local desktop application.

**Outgoing:** None — the only outbound HTTP call at runtime is the Stockfish GitHub API query (Windows install only, described above).
