## `BoardState.board_ref` — Read-Only Contract (Codex MEDIUM)

The `board_ref` property exposes the live underlying `chess.Board`. Its docstring MUST contain the following verbatim contract per Codex MEDIUM:

```
Returns the live `chess.Board` for read-only access by controller/view layers.
Callers MUST NOT mutate (no `push`, `pop`, `set_fen`, etc.).
Use `board` (snapshot copy) when mutation is possible.
```

This wording is load-bearing — Codex flagged the property as a "maintainability hazard if not tightly policed." Misuse of `board_ref` is a regression that a reviewer should reject. The contract is documented in:
1. The property's docstring (verbatim text above).
2. This plan's `must_haves.truths` (greppable invariant).
3. `tests/test_board_state_terminal.py::TestBoardRefProperty::test_board_ref_docstring_contains_readonly_contract` (asserts the docstring contains the words "MUST NOT mutate").

A future enhancement may add a runtime guard (e.g., wrapping `_board` in a proxy that blocks `push`/`pop`), but that is out of scope for Phase 1.

## `board_ref` Adoption Sites (Codex MEDIUM audit)

This plan converts the following read-only call sites in `openboard/controllers/chess_controller.py` from `.board` to `.board_ref`. Each site is a hot path that does not mutate the returned board:

| Site | Method | Why convert |
|------|--------|-------------|
| `announce_attacking_pieces` (lines ~462-511) | reads `board.attackers(...)` and `board.piece_at(...)` | pure read; called on every "what attacks this square?" query — high frequency. |
| Any other read-only `.board` use that performs only `piece_at`, `attackers`, `is_attacked_by`, `is_check`, `is_checkmate`, `fen()` calls | (audit during Task 2) | identical semantics — eliminates the snapshot cost. |

Sites that MUST keep using `.board` (snapshot semantics required):
- `announce_last_move` (after Plan 01) — uses `board.copy(); board.pop()` to roll back one move; mutation is intentional, requires a copy.
- Any test seed that expects state isolation across calls.
